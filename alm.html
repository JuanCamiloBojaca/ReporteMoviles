Permisos

Al crear una nueva aplicación en Flutter siempre se requiere del permiso de internet ya que este es necesario para el correcto funcionamiento de a app. Específicamente necesita tener permisos de red para poder comunicarse con el servidor y extraer y manipular los datos del usuario.


Adicionalmente, se está haciendo uso de la ubicación del usuario para cambiar el idioma dependiendo del sitio donde se esté usando. Esto se realiza agregando la dependencia asociada a la geolocalización (flutter_localization) y la dependencia para la traducción (intl_translation). Estas dependencias permiten conocer la ubicación del usuario aun sin el permiso de este. Sin Embargo haciendo un poco mas de zoom en las mismas se puede notar que para el que extiende la clase solo está disponible la información del código del país por lo que no es un fallo de seguridad importante.

Almacenamiento

Interno

Para el almacenamiento en local se usa la librería shared preferences. Con la cual se almacenan tres cosas, saber si el usuario se encuentra logeado, saber si ya calificó la app y saber si tiene activados las analíticas. Cabe aclarar que solo mantiene el estado de di esta loggeado o no de lo contrario sería un gran problema de seguridad.

Externo

Aparte de usar cache para almacenar algunas imágenes de manera temporal, el resto de los datos de la aplicación que cargan y actualizan de un servidor remoto. A excepción de la autentificación el código para conexión a datos externos se encuentra centralizado en una única clase que provee un punto de acceso para que las vistas puedan consumir los datos.

La aplicación se comunica con 4 repositorios de datos diferentes, firbase en donde se maneja la lógica del negocio, todo lo que tiene que ver con los usuarios y su historial. UNTAPPD y BREWERY que son dos Apis que permiten poseen el registro de las cervezas y cullas implementaciones sobre una interfaz mas grande se encuentra en el código. Finalmente, sentry que es para subir reportes de error de la aplicación. Todos los anteriores requieren de llaves para su correcto funcionamiento, estas llaves no se encuentran en el repositorio y solo se da un archivo de ejemplo de donde irían.
